# Решение ошибки 403 Forbidden от Claude API

## Проблема
Ошибка `403 Forbidden: Request not allowed` при обращении к Claude API через прокси.

## Возможные причины

1. **IP прокси заблокирован Anthropic** - некоторые прокси могут быть в черном списке
2. **Неправильная настройка прокси** - прокси не работает корректно
3. **Проблемы с API ключом** - ключ не имеет прав или истек
4. **Блокировка региона** - прокси из определенного региона может быть заблокирован

## Решения

### 1. Проверка без прокси

Временно отключите прокси в `config.py`:

```python
CLAUDE_PROXY = os.getenv("CLAUDE_PROXY", "")  # Пустая строка
```

Запустите бота и проверьте, работает ли API без прокси.

### 2. Проверка прокси

Выполните на сервере:

```bash
cd /opt/bot
source venv/bin/activate
python test_claude_proxy.py
```

Этот скрипт проверит работу API с прокси и без.

### 3. Попробуйте другой прокси

Если текущий прокси не работает:
- Попробуйте SOCKS5 прокси вместо HTTP
- Используйте другой прокси сервер
- Проверьте, что прокси не заблокирован

### 4. Проверка API ключа

1. Зайдите в [Anthropic Console](https://console.anthropic.com/)
2. Проверьте статус API ключа
3. Убедитесь, что ключ активен и имеет права доступа
4. Проверьте лимиты использования

### 5. Альтернативные решения

Если прокси не работает:

**Вариант А: Использовать VPN на сервере**
```bash
# Установить VPN клиент на сервере
# Настроить автоматическое подключение
```

**Вариант Б: Использовать другой прокси провайдер**
- Выберите прокси с IP из разрешенных регионов
- Используйте резидентные прокси вместо дата-центровых

**Вариант В: Запустить бота на сервере без блокировки**
- Используйте VPS в регионе, где Anthropic доступен
- Например, США, Европа

## Диагностика

Проверьте логи бота:

```bash
sudo journalctl -u telegram-bot.service -f
```

Ищите сообщения:
- `[INFO] Claude клиент создан с прокси:` - прокси используется
- `[INFO] Claude клиент создан без прокси` - прокси не используется
- `⚠️ Ошибка настройки прокси:` - проблема с настройкой прокси

## Быстрое решение

Если нужно срочно запустить бота:

1. Временно отключите прокси в `config.py`:
   ```python
   CLAUDE_PROXY = ""
   ```

2. Перезапустите бота:
   ```bash
   sudo systemctl restart telegram-bot.service
   ```

3. Проверьте работу API

4. Если работает без прокси - проблема в прокси, попробуйте другой
