# 🚀 Быстрая установка Vosk модели

## Проблема
Бот не может распознать речь, потому что не установлена модель Vosk.

## ✅ Решение

### Вариант 1: Автоматическая установка (Рекомендуется)

```bash
# На сервере или локально
python install_vosk_model.py
```

Скрипт автоматически:
1. Создаст папку `vosk_models/`
2. Скачает модель (50 MB или 1.5 GB)
3. Распакует модель
4. Готово!

---

### Вариант 2: Ручная установка

#### На сервере (Linux):

```bash
cd /opt/bot

# Создать папку
mkdir -p vosk_models
cd vosk_models

# Скачать маленькую модель (50 MB)
wget https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip

# Распаковать
unzip vosk-model-small-ru-0.22.zip

# Удалить ZIP
rm vosk-model-small-ru-0.22.zip

# Вернуться в корень
cd ..
```

#### На Windows:

1. Создай папку `vosk_models` в папке с ботом
2. Скачай модель: https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip
3. Распакуй ZIP в папку `vosk_models/`
4. Должно быть: `vosk_models/vosk-model-small-ru-0.22/`

---

## 📦 Какую модель выбрать?

### Маленькая (50 MB) - Рекомендуется для начала
- ✅ Быстрая загрузка
- ✅ Хорошее качество
- ✅ Подходит для большинства задач
- Ссылка: https://alphacephei.com/vosk/models/vosk-model-small-ru-0.22.zip

### Большая (1.5 GB) - Для лучшего качества
- ✅ Лучшее распознавание
- ⚠️ Долгая загрузка
- ⚠️ Больше места на диске
- Ссылка: https://alphacephei.com/vosk/models/vosk-model-ru-0.42.zip

---

## ✅ Проверка установки

После установки проверь:

```bash
# Проверить наличие модели
ls -la vosk_models/

# Должно быть:
# vosk-model-small-ru-0.22/  (или vosk-model-ru-0.42/)
```

---

## 🔄 Перезапуск бота

После установки модели:

```bash
# На сервере
sudo systemctl restart telegram-bot.service

# Или локально
# Просто перезапусти бота
```

## ❓ Проблемы?

### Модель не загружается:
1. Проверь путь: `vosk_models/vosk-model-small-ru-0.22/`
2. Проверь права доступа: `chmod -R 755 vosk_models/`
3. Проверь логи бота

### Все еще не работает:
1. Убедись, что установлен Vosk: `pip install vosk`
2. Проверь версию Python (нужен 3.7+)
3. Проверь логи на ошибки
